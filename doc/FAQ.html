1. What is GeekOS?

GeekOS is an instructional operating system.  It is intended to be used in undergraduate operation system courses, or for self-study.  It implements a very minimal set of features on top of bare x86 PC hardware.  In a series of projects included in the GeekOS distribution, students add important operation system features such as processes, virtual memory, a filesystem, and interprocess communication.  Once all of the projects have been completed, GeekOS resembles a very simple version of Unix.

2. What do I need to run GeekOS?

GeekOS runs mainly on the Bochs PC emulator.  Running GeekOS on an emulator has several important advantages over running it on a real computer:

  1. The emulator boots very quickly (around 1 second), allowing a much quicker compile/edit/debug cycle
  2. The logfile produced by bochs (bochs.out) contains very useful information for diagnosing crashes
  3. Recent versions of Bochs (2.0 and later) can be build with stubs for the gdb debugger, allowing easy debugging of the kernel

Please note that although GeekOS is tested on real hardware from time to time, you will probably need to do some hacking to get it working on your PC.  Submissions of bug fixes that make GeekOS work on more hardware are gladly accepted.

To compile GeekOS from source, you will need the following software:

  • gcc targeting i386/ELF; the default compiler on Linux and FreeBSD systems will work, and cross compilers targeting i386-elf should also work
  • A host platform C compiler supporting ANSI C
  • The Netwide Assembler (NASM)
  • GNU Make
  • Perl, version 5.0 or later
  • AWK, egrep

3. Why did you write another instructional operating system?

Good question.  There are a lot of other good instructional operating systems out there: Minix, Topsy, OS/161, and Nachos, to name a few.

GeekOS pursues slightly different design goals than any of the above.  I wanted to combine simplicity with realism.  Simplicity means that GeekOS tries to use the simplest algorithm or data structure that will work; for example, you will find lots of linked lists and arrays in GeekOS, but no hash tables or balanced trees.  Realism means that GeekOS resembles a real operating system.  When the projects are completed, GeekOS has processes, paged virtual memory, and a hierarchical filesystem.  Adventurous hackers can boot GeekOS on a real PC.

4. Why does GeekOS suck so much?

Many of the implementation decisions made in GeekOS would be unacceptable in a real operating system.  For example:

  • No support for PCI devices
  • IDE disk driver uses CHS addressing and programmed I/O
  • IDE requests proceed synchronously with busy waiting
  • Page stealing does a complete walk of every page of memory with interrupts disabled
  • Executable loading copies entire program image into memory twice before process starts
  • And so on...

Keep in mind that the main goal of GeekOS is simplicity.  If you want to fix some of the more egregious limitations in GeekOS, in a way that maintains or improves the simplicity of the system, please submit a patch.

5. Will GeekOS run program X?

GeekOS does not conform to any existing API standard, and has only a very minimal C library.  Therefore, any program written for another operating system will require considerable porting effort.
